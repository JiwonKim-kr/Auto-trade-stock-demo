# AI 자동매매 시스템 — 설계 결정 문서 (에이전트 핸드오프용)

> **이 문서의 목적**: 이 프로젝트를 이어받는 에이전트가 대화 맥락 없이도
> "무엇을, 왜 그렇게 정했는지"를 이해하고 작업을 이어가도록 하는 컨텍스트 문서.
> 결정만이 아니라 **기각된 대안과 그 이유**, **작업 시 지켜야 할 불변 원칙**을 포함한다.
> 코드 골격은 `event_drift_system/` 패키지에 있으며 이 문서와 짝을 이룬다.

---

## 1. 프로젝트 의도와 범위

- 개인 주식투자·연구 목적의 **시그널 기반 자동매매 시스템**.
- 대상 시장: **한국 주식시장**. 계좌/실행은 **토스증권 Open API** 전제.
- 목표: 알고리즘의 매매 성능을 **통계적으로 판단 가능한 형태**로 측정하고,
  벤치마크 대비 초과수익(알파)이 있는지 검증한다.
- 현재 단계: **백테스트 골격 완성**. 실데이터·브로커 연결·검증은 미완(§6, §7).

---

## 2. 최종 전략 (한 문단 요약)

개별기업 이벤트(실적 서프라이즈에 따른 표류 PEAD, 뉴스 감성 등)를 알파 소스로,
거시·지정학 이벤트는 노출을 줄이는 리스크 필터로 사용하는 **롱온리·이벤트 표류
전략**. 보유기간은 대략 **1~4주(고정 아님, 청산 규칙이 결정)**, 종목은 **20~40개**를
시차 진입해 표본 수와 연속 손익곡선을 확보한다. 비용 대비 기대수익이 충분할 때만
진입하고(**기대이동폭 ≥ 라운드트립 비용 ×3.5**), 포지션은 **총자산의 50%를 매 회
재계산하는 대칭 방식**으로 배분하며, 별도 서킷브레이커로 하방을 막는다.

---

## 3. 설계 결정 로그 (결정 → 이유)

### 3.1 자금·리스크

- **대칭 포지션 사이징**: 운용자본 = 총자산 × 50% × 레짐배수를 **매 리밸런싱마다
  재계산**. 손실 시 노출이 자동 축소, 이익 시 확대 — 한쪽으로 래칫되지 않는다.
  - _배경_: 초기 아이디어("현금 50% + 늘어난 현금의 50%를 추가")는 **이익 방향만
    키우고 손실 시 줄이지 않는 비대칭 래칫**이었다. 연승 직후 노출이 최대가 되어
    평균회귀 지점에서 가장 위험. 이를 대칭 재계산으로 교체함.
- **"50% 운용"은 손실 상한이 아니다**: 매도 후 재투입을 반복하면 누적 손실이 50%를
  넘을 수 있다. 50%는 순간 노출 상한일 뿐. → **별도 서킷브레이커 필수**.
- **서킷브레이커**: 일일 손실 -5% 또는 누적 MDD -15% 도달 시 **신규 진입 중단**
  (기존 포지션 청산은 계속 허용).
- **종목당 최대 비중 6%**: 단일 종목 집중 위험 차단.
- **실현손익 기준 사이징**: 미실현(평가) 이익으로 베팅을 키우지 않는다.

### 3.2 보유기간·표본

- **보유 1~4주, 고정 상수 아님**: 실제 보유일은 청산 규칙(손절/타임스톱/시그널
  소멸)이 결정. 위 범위는 "백테스트 평균 보유가 대략 이 안에 들도록" 하는 설계 목표.
- **성능 판단의 기준은 '잦은 매매'가 아니라 '독립 베팅 수'**: 회전을 빠르게 하는
  대신 **종목 폭(breadth) 20~40개 + 시차 진입**으로 베팅 수를 번다. 이러면
  개별 거래는 느려도 포트폴리오 손익곡선은 매일 갱신되어, 보유기간과 성능
  피드백 주기가 분리된다.
- **판단 게이트**: 독립 트레이드 **100건 미만이면 "판단 보류"**. 샤프의 표준오차
  SE ≈ √((1+0.5·SR²)/N) 로 계산 — N이 작으면 운/실력 구분 불가.

### 3.3 시그널·이벤트

- **개별기업 이벤트 = 알파 소스**: 실적·가이던스·수주·수급·뉴스는 매일 다수 발생
  → 표본 충분, 1~4주 표류 효과 존재(PEAD 등). 시스템의 수익 원천.
- **거시·지정학 이벤트(전쟁·금리·규제) = 리스크 필터**: 10년에 몇 번뿐 →
  통계 검증 불가·비정상성. 알파로 쓰지 말고 "변동성/긴장 高 국면엔 노출 축소"라는
  **방어 스위치**로만 사용. 예측이 아니라 대응.
- **헤드라인 방향 ≠ 가격 방향**: 선반영("buy the rumor, sell the news") 때문에
  뉴스 감성 시그널은 약한 가중으로 취급.
- **이벤트와 펀더멘털은 대체가 아니라 보완**: 이벤트=촉매(언제), 펀더멘털=방향·크기(왜/얼마나).

### 3.4 비용·평가

- **비용 인지 진입 필터**: 기대이동폭 ≥ 라운드트립 비용 × 3.5 일 때만 매수.
  현재 파라미터로 라운드트립 비용 ≈ 0.63% → 진입 최소 기대이동폭 ≈ 2.2%.
- **모든 평가는 넷(net) 수익률로**: 백테스트의 모든 체결은 비용모델을 통과.
- **성능 지표는 절대수익이 아니라 벤치마크 대비 알파 + 위험조정수익(Sharpe/MDD)**:
  롱온리·장기보유는 수익 대부분이 시장 베타에서 나오므로 절대수익은 오해 소지.

## 4. 기각된 대안 (에이전트가 다시 제안하지 말 것)

| 기각안                                            | 기각 이유                                                                                                                    |
| ------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------- |
| **소액주 초단타(분/초 단위 회전)**                | 라운드트립 비용(매도세+수수료+스프레드)이 소액 차익을 초과. 2026년 증권거래세 부활로 더 악화. 그로스 +여도 넷 −가 되기 쉬움. |
| **비대칭 사이징(이익에만 베팅 확대)**             | 손실 시 디레버리징이 없어 위험이 단조 증가. §3.1의 대칭 방식으로 대체됨.                                                     |
| **수개월+ 장기보유 단독 전략**                    | 독립 베팅 수가 급감해 성능의 통계 검증이 사실상 불가능. 종목 폭으로 보완하는 중기(1~4주) 전략으로 절충.                      |
| **거시 이벤트(전쟁 등)를 트레이딩 시그널로 사용** | 희소·비반복·비정상성으로 백테스트가 사후 짜맞추기가 됨. 리스크 필터로만 사용.                                                |
| **절대수익만으로 성능 평가**                      | 시장 베타를 알파로 착각. 벤치마크 대비 초과수익으로 평가.                                                                    |

## 5. 시장·제도 제약 (한국, 2026 기준)

- **증권거래세 부활(2026.1.1~)**: 코스피 0.05% + 농특세 0.15% = 0.20%, 코스닥 0.20%.
  **손절(손실 매도)에도 부과**되며 매도 빈번할수록 부담 증가 → 단타 불리의 핵심 근거.
- **가격제한 ±30%, 거래정지·관리종목·상장폐지** 이벤트로 청산 자체가 막힐 수 있음.
- **T+2 결제**: 매도 대금은 2영업일 후 가용. 즉시 재투자 가정 금지(미수 방지).
- **호가단위·유동성**: 소액주는 스프레드·시장충격 큼 → 슬리피지 보수적 가정 필요.
- 세율 등 제도 수치는 변동 가능하므로, 실거래 시점에 **재확인**할 것.

## 6. 현재 구현 상태 (`event_drift_system/` 패키지)

**실제 구현됨(설계 핵심 로직):**

- `config.py` — 모든 튜닝 파라미터
- `portfolio.py` — 대칭 사이징, 비용 필터, breadth 기반 목표 포지션
- `risk.py` — 청산 규칙 + 서킷브레이커
- `execution.py`(CostModel) — 한국 2026 비용 모델
- `evaluation.py` — Sharpe/MDD/알파/샤프 표준오차/표본 게이트
- `regime.py` — 거시/변동성 → 노출배수

**스텁(연결 필요):**

- `data_feeds.py` — PriceFeed/EventFeed/Universe, **토스 어댑터**
- `execution.py`(TossBroker) — 주문·예수금 연결
- `signals.py` — 시그널 로직은 있으나 **실데이터 파라미터 검증 미완**
- `backtest.py` — 루프 골격은 있으나 거래일 캘린더·다음영업일 체결·T+2 원장 미완

`python3 main.py` 로 배선·데모 동작 확인 가능.

## 7. 열린 질문 / 다음 단계 (우선순위)

1. **데이터 파이프라인**: point-in-time 이벤트 로그 + 시점별 유니버스(상폐 포함,
   생존편향 제거) + 재작성 안 된 재무데이터 확보.
2. **백테스트 현실화**: 거래일 캘린더 도입, **시그널 다음 영업일 체결**(같은 날
   종가 진입은 미래정보 누출), **T+2 현금원장**.
3. **시그널 검증**: 각 시그널의 표류 효과를 아웃오브샘플/워크포워드로 확인.
   유효하지 않으면 폐기 — 여기서 대부분의 후보가 탈락하는 게 정상.
4. **토스 어댑터 구현**: 레이트리밋·장운영시간·가격제한·거래정지·최소호가단위 처리.
5. **호라이즌·시그널 계열 확정**: 아직 미확정. 모멘텀/반전/이벤트·수급/뉴스 감성 중
   어디에 무게를 둘지 결정 필요.

## 8. 작업 시 지켜야 할 불변 원칙 (에이전트용 가드레일)

1. **비용은 항상 반영한다.** 어떤 성능 수치도 비용·슬리피지 차감 후(넷)로만 보고할 것.
2. **미래정보 금지.** 결정 시점에 알 수 있던 데이터만 사용. 다음 영업일 체결 원칙 유지.
3. **생존편향 금지.** 오늘의 종목 리스트로 과거를 백테스트하지 말 것.
4. **표본이 부족하면 판단하지 말 것.** 독립 트레이드 <100이면 "판단 보류"로 보고.
5. **거시 이벤트는 알파가 아니라 필터.** 전쟁·금리 등을 예측 시그널로 만들지 말 것.
6. **사이징은 대칭.** 손실 시 노출을 줄이는 로직 없이 이익 방향만 키우지 말 것.
7. **서킷브레이커를 무력화하지 말 것.** 신규 진입 차단 시에도 청산은 허용.
8. **평가는 벤치마크 대비.** 절대수익만으로 성공을 주장하지 말 것.
