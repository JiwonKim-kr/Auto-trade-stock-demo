# Self-Evaluation — 토스 AI 자동매매 (2026-07-03 기준)

> 구현 주체(Claude)가 수행한 프로젝트 자기 평가 기록. 근거는 코드·테스트·시뮬레이션 실측.
> 평가는 정직을 우선한다 — 잘된 것과 미완/약점을 같은 무게로 적는다.

## 1. 산출물 인벤토리 (초기 목표 대비)

| 목표(TECH-STACK 킥오프) | 상태 |
|---|---|
| 토스 클라이언트(함정 내재화)·스모크 우선 방법론 | ✅ 실응답 픽스처 회귀 고정 |
| DRY_RUN 주문층 + 가드레일 + 킬스위치 | ✅ +서킷브레이커·결정적 청산(계획 초과) |
| 유니버스→스크리너→LLM 판단 연결 | ✅ +조사(web_search)·레짐·비용 게이트·ADV 선별 |
| DB 영속화 | ✅ SQLite/PG 겸용, 엔진 상태 재시작 생존 |
| 리컨실 | ✅ LIVE 불일치 = 자동 거래 중단 |
| 성과 측정 | ✅ 페이퍼 P&L(자기일관 루프) + 평가(표본 게이트) + 캘리브레이션 + 자동 보고서 |
| 토스 없는 검증 환경 | ✅ 스트레스 샌드박스 + 리플레이 백테스트 |
| GCP 배포(M2) / LIVE executor(M3) / 데스크톱 | ❌ 미착수(계획서 §3·§4 스펙 완비) |

**정량**: 테스트 263개(전부 통과) · 신규 모듈 ~25개 · LLM 실 API 의존 테스트 0(전부 주입/목).

## 2. 안전성 — 평가: **강함(페이퍼 기준 검증됨), LIVE 는 미개통이 곧 안전**

**검증 근거(스트레스 샌드박스 실측, 2026-07-03)**:
- 전 방어층 활성: 시장 **−25% 갭·−7%×10 폭락에서 포트폴리오 MDD ≤ 8.0%**
  (레짐 ×0.5 → 손절 −8% → 비중 10% 층별 흡수).
- 최후 방어선 단독: CB **−15% 돌파 즉시 발동 + 이후 신규 매수 0**(불변식 테스트로 CI 고정).
- 구조적 0: DRY_RUN 에서 executor 호출 자체가 없음(주문 전송 코드 미작성 — 의도).

**이 과정에서 발견·수정한 결함(자기 평가의 핵심 증거 — 장치가 실제로 일함)**:
1. 콜드스타트 사이징 버그 — 빈 장부에서 첫 매수 영구 불가(비중 분모 오류). **샌드박스가 발견**.
2. LIVE 가 DB 없이 켜짐 → 일일 한도 틱마다 리셋. **점검이 발견**, 강제 강등으로 해소.
3. 청산이 LLM 재량뿐 → 결정적 손절·타임스톱 신설.
4. SQLite tz 미보존 → 리컨실 창 9시간 시프트 잠재 버그. 일괄 정규화.

**남은 안전 갭(정직)**: 입출금-서킷브레이커 왜곡(§1.3, LIVE 전 필요) · 미체결/부분체결 리컨실
오탐(fills 연동 전 보수 오탐으로 수용) · 페이퍼 모드에서 CB 일일손실 트리거 비활성(낙폭만 동작).

## 3. 효용성 — 평가: **측정 장치는 완비, 알파는 미검증(정직한 현재 위치)**

- 알파 가설(모멘텀 스크리너 × LLM 뉴스 확인)은 합리적이나 **검증 전**. 이 프로젝트의 강점은
  수익 주장이 아니라 **"수익 주장을 검증할 규율"이 코드로 존재**한다는 것: 넷 손익 강제,
  표본 게이트(N<100 판단 보류), 벤치마크 대비, confidence 캘리브레이션(단조성 판정), 보수적
  체결/오탐 방향.
- **LLM 경로는 아직 실 API 로 1회도 미검증** — `ANTHROPIC_API_KEY` 대기(스크립트 준비됨).
  현재 페이퍼가 돌면 결정적 폴백 기준 데이터가 쌓인다(그 자체로 베이스라인 가치).
- 리플레이는 LLM 소급 평가가 원리적으로 불가함을 명시(훈련데이터 look-ahead) — 파라미터
  상대 비교 전용으로 범위를 못박음.

## 4. 아키텍처·코드 품질 — 평가: **원칙이 일관되게 지켜짐**

- 불변 원칙 8개(계획서 §0)가 실제 코드에서 관철: LLM 은 방향+confidence만 / 하드룰은 바깥 /
  매도는 안 막음 / Decimal / DB·I/O 는 경계 / 주입형 확장(entry_gate·regime·forced_exits·
  holdings·cash — run_tick 시그니처가 그 역사).
- 주입형 설계의 배당: FakeToss→SyntheticToss→ReplayToss 로 **동일 파이프라인**이 테스트·
  스트레스·백테스트에 재사용됨(코드 분기 0).
- 약점: `api/tick.py` 오케스트레이션이 비대(조립 단계 10+) — M2 전 함수 분할 여지.
  Alembic 부재(스키마 변경 시 로컬 DB 재생성 전제 — 배포 전 도입 필요).

## 6. 즉시 실행 항목 (사용자 액션 포함)

1. **페이퍼 운용 시작**(데이터가 유일한 재판관): `.env` 4줄 + `run_local.py` — 오늘부터.
2. `ANTHROPIC_API_KEY` 투입 → `llm_live_check.py` 로 LLM 경로 최초 실검증.
3. `data/krx_holidays.json` 2026 목록 **KRX 공지로 검증**(현재 예상치 표기).
4. 증권거래세율 실확인(현재 0.15% 가정 — 보수 방향이나 캘리브레이션 뿌리).
5. 텔레그램 봇 토큰 설정(무인 운용 전제조건).
